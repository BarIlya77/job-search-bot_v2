## ✨ Возможности

- 🔍 **Поиск вакансий** – по заданным фильтрам (профессия, зарплата, опыт, график, занятость, город)
- 📋 **Гибкая настройка фильтров** – через inline‑кнопки или ручной ввод
- ⏰ **Автоматический поиск** – планировщик проверяет новые вакансии и присылает уведомления
- 💾 **Сохранение вакансий** – добавление в избранное (в разработке)
- 📝 **Генерация сопроводительных писем** – через DeepSeek AI (в разработке)
- 📊 **Статус и статистика** – просмотр текущих фильтров и состояния планировщика
- 🧠 **Чистая архитектура** – разделение на слои (бот, сервисы, хранилище)

## 🛠 Технологии

- **Python 3.10+** – основной язык
- **python-telegram-bot 22.5** – взаимодействие с Telegram API
- **aiohttp** – асинхронные запросы к HH.ru
- **SQLAlchemy 2.0 + SQLite/PostgreSQL** – работа с БД (асинхронно)
- **APScheduler 3.11** – планировщик задач
- **DeepSeek API** – генерация текстов (опционально)
- **python-dotenv** – управление переменными окружения
- **alembic** – миграции БД (при переходе на PostgreSQL)

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/job_search_bot_v2.git
cd job_search_bot_v2
```

### 2. Настройка окружения
Создайте файл .env на основе .env.example:

```bash
cp .env.example .env
Отредактируйте .env, указав свои данные:

BOT_TOKEN=your_telegram_bot_token
DEEPSEEK_API_KEY=your_deepseek_api_key   # опционально
DATABASE_URL=sqlite+aiosqlite:///./data/jobs.db
SCHEDULER_ENABLED=true
CHECK_INTERVAL=60
LOG_LEVEL=INFO
```

### 3. Установка зависимостей

Рекомендуется использовать виртуальное окружение:

```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows
pip install -r requirements.txt
```

### 4. Запуск бота

```bash
python main.py
При первом запуске автоматически создастся база данных SQLite в папке data/.
```

## 📱 Использование

### Основное меню

После команды `/start` бот покажет главное меню с кнопками:

- **🔍 Поиск вакансий** – запускает поиск по текущим фильтрам
- **⚙️ Фильтры** – настройка параметров поиска
- **📊 Статус** – просмотр текущих фильтров и состояния планировщика
- **❓ Помощь** – список команд

### Настройка фильтров

В меню фильтров можно задать:

- **Профессия** – выбор из предложенных или свой вариант
- **Зарплата** – минимальная сумма в рублях
- **Опыт работы** – без опыта, 1–3 года, 3–6 лет, более 6 лет
- **Формат работы** – офис, удалённо, гибрид, гибкий график
- **Тип занятости** – полный день, частичная, проектная, стажировка
- **Город** – выбор из популярных или свой вариант (название будет преобразовано в ID)

Все настройки сохраняются в базе данных и используются при поиске.

### Просмотр вакансий

После нажатия «🔍 Поиск вакансий» бот выводит вакансии по одной, с кнопками навигации:

- ◀️ Назад / Вперед ▶️ – листание страниц
- 💾 Сохранить – добавить в избранное (в разработке)
- 👎 Скрыть – исключить вакансию из показа (в разработке)
- 📝 Письмо – сгенерировать сопроводительное письмо (в разработке)

## ⏰ Планировщик

Планировщик автоматически проверяет новые вакансии для всех активных пользователей с заданным интервалом (по умолчанию **60 минут**). При обнаружении новых вакансий бот отправляет уведомление с тремя самыми свежими предложениями. Если вакансий больше трёх, предлагается воспользоваться ручным поиском через главное меню.

### Как это работает

1. Планировщик запускается при старте бота (если `SCHEDULER_ENABLED=true` в `.env`).
2. Каждые `CHECK_INTERVAL` минут выполняется проверка для всех активных пользователей.
3. Для каждого пользователя загружаются его фильтры и выполняется поиск вакансий за последние **24 часа**.
4. Найденные новые вакансии (которые ещё не отправлялись этому пользователю) отправляются в чат.
5. Чтобы избежать спама, отправляется не более **3 вакансий** за один раз.

### Управление планировщиком через бота

- `/scheduler_start` – запустить (можно указать интервал, например `/scheduler_start 30`).
- `/scheduler_stop` – остановить.
- `/scheduler_interval 15` – изменить интервал на **15 минут**.
- `/scheduler_status` – показать состояние: запущен ли, интервал, количество пользователей и т.д.
